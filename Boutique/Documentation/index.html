Requirements

Folder Structure

  • Woocommerce File: This folder contains a file which we will use later to configure our Woocommerce store
  • Documentation: Contains the documentation for this app

Getting Started

Before starting you need to Install Ionic Framework on your computer, to do that use NPM (If you don't have NodeJs installed download it from here)

Then open up your terminal and run the following command:

  npm install -g cordova ionic

You now have Ionic framework installed on your computer.

 

Now, you need to make sure that you have the required SDK to build your mobile app.

Setting up IOS and Android SDK is well documented on the Apache Cordova website:

  • Android
  • IOS

 

Now that we have all the requirements for building mobile apps with ionic, we can start by creating our project.

Open up the terminal and navigate to the folder you want to create your app in, then run this command

* Always replace AppName with the name of your App

ionic start AppName blank
Now navigate to the app folder

cd AppName

Then we add IOS and Android platforms (only add the one you're planning to use):

ionic platform add ios
ionic platform add android

Copy and replace the dowloaded www folder with the one in the AppName folder.

Then we need to add the Cordova inApp browser plugin:

  1. Make sure your in AppName folder
  2. Run the following command
cordova plugin add cordova-plugin-inappbrowser

 

Now, we're good to go, you can find more about creating a project with Ionic on their official documentation.

Setting up

API configuration

Let's start by setting up our API configuration:

  1. Login to your Wordpress admin dashboard
  2. Go to Woocommerce -> Setting
  3. Select the API  tab
  4. Make sure Enable the REST API is checked
  5. Click on Keys/Apps
  6. Click on Add Key
  7. Then Generate Key
  8. Make sure to copy these keys in a safe place, you won't be able to see them again in your dashboard

Configuring Woocommerce

Now we need to configure Woocommerce by a simple Copy/Paste operation.

  1. First copy the file class-wc-api-products.php from the directory Woocommerce File inside the downloaded files.
  2. Use your favorite FTP client to access your website files.
  3. Now paste the copied file in \wp-content\plugins\woocommerce\includes\api directory and click on Replace REMEMBER TO BACKUP YOUR FILE
That should do it, but if you don't have FTP access follow these steps (you can skip them if you successfully finished the steps above):
  1. First open up the Woocommerce File\class-wc-api-products.php file with a text editor and copy all its content (ctrl+a then ctrl+c in Windows and command+a then command+c on a Mac)
  2. Now open up Wordpress plugin code editor from your dashboard (Plugins/Editor)
  3. Select the Woocommerce plugin in the Select plugin to edit list and click 'Select'
  4. In the sidebar you can see the plugin files, click on the one that's called woocommerce/includes/class-wc-api.php
  5. Click on woocommerce/includes/api/class-wc-api-products.php (you can search for it by using Ctrl+f of Command+f)
  6. Now you should see the content of the class-wc-api-products.php file in your page, all you need to do is simply select all the content of this file and delete it and then paste the previously copied content from our file (Woocommerce File\class-wc-api-products.php)
  7. Click 'Update File' button. Wordpress should tell you that the file was edited successfully.

App Configuration

It's time to generate our app configuration, you can do it easily with our generator or you can do it manully.

Using the config generator:
  1.  First visit Themeleger.com/boutique
  2. Click on Generate Configuration
  3. Fill with the informations of your App 
    		Store Title: The title displayed in the header bar of the app
		Store Link: your store homepage URL
		Checkout Link: your store checkout URL
		Customer Key & Customer Secret: The keys generated when setting up the API
		Timeout: timeout when calling the server before throwing an error (default 20000 ms)
		Messages: you can change the messages displayed to the user or you can leave it to default
  4. Click on Generate, this should download your new config.js file
  5. Replace www/js/config.js file with the downloaded one (always backup before replacing)

 

Generation the configuration file

 

Manually changing configuration (ignore this section if you successfully generated the config.js file) :
  1. Open www/js/config.js file with a text editor
  2. Change the value of the config object
var config = {

    title: 'Boutique', // Title displayed in the app

    link: 'http://192.168.1.3/wp/', // Woocommerce homepage URL

    checkout: 'http://192.168.1.3/wp/index.php/checkout/', // Woocommerce checkout URL

    timeout: 20000, // Timeout of api calls in milliseconds

    api: {

        link: 'http://192.168.1.3/wp/'+'wc-api/v3/', // API link

        key: 'ck_ab881ffc26e2175cac94013c5276ff38be129a2c', // API customer key 

        secret: 'cs_83b1c7248210865e26856a29974200ee17bbf881', // API customer secret

        signature: 'HMAC-SHA1', // API signature algorithm

        version: '1.0' // OAuth Version

    },

    messages: { // HTML Allowed

        sale_text: 'Sale', // Text on the sale badge

        added_cart: 'Successfully added to your cart !', // Message to show when a product is added to cart

        exists_cart: 'Product already exist in your cart !', // Message to show when a product already exists in cart

        server: 'There was a problem connecting to the server !<br>Please try again later', // Message to show if there's no server connection

        no_internet: 'Please check your internet connection !', // Message to show if there's no internet connection

        empty: "There's no products in this categorie.", // Message to show if a categorie is empty

        empty_cart: "There's no item in your cart !", // Message to show if the user cart is empty

        empty_shop: "The shop is empty", // Message to show if the website have no categories

        error: 'There was an error' // Generic error message

    },

    views: { // Views title

        homePage: 'Boutique',

        cart: 'Cart'

    }

};
 
 

Testing

You can test your app in the browser, on an emulator or on your device:

Testing with browser

  1. Navigate to the AppName folder with your terminal 
  2. Execute the following command (Resize your browser to simulate a smartphone)
ionic serve
The checkout page only works on real devices or emulators

 

Testing with an emulator

  1. You can use the Iphone emulator in XCode or the android studio emulator (it's better to use genymtion emulator for android which is alot faster) 
  2. For Ios run the following:
ionic build ios
ionic emulate ios

and for Android:

ionic run android

Testing with a device

Use the same commands as in an emulator but with a device plugged in.

For android make sure the USB debugging is enabled on your device

Find more about testing ionic app here.

Changing the style

You can change the appearance of your app by using one of our themes or by creating your own theme.

Changing the theme

  1. Simply copy and replace the style.css file from the theme folder in www/css/style.css (make sure you have a backup file before replacing)
More themes will be available in the futures.

Creating your own theme

You can create your own theme with SASS.

  1. Open up SCSS/style.scss with a text editor
  2. Edit the code (colors, fonts, height and width, etc..)
  3. Compile the file using a SASS compiler (recommended KOALA or GULP)
